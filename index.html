Actor entities — ผู้ใช้งานระบบ
entity · actor
Account

เก็บข้อมูลผู้ใช้ที่ล็อกอินผ่าน Google OAuth รวมข้อมูลธนาคารสำหรับรับเงินชดเชยเคลม และ is_company บอกว่าเป็นบุคคลธรรมดาหรือนิติบุคคล

Methods & คำอธิบาย
login()รับ Google token ตรวจสอบกับ DB แล้วสร้าง JWT ส่งกลับ ถ้าไม่มีในระบบจะสร้าง Account ใหม่ให้อัตโนมัติ
logout()ลบ session / ทำให้ JWT ใช้ไม่ได้อีก
updateProfile()แก้ไขข้อมูลส่วนตัว เช่น เบอร์โทร ชื่อบัญชีธนาคาร เลขบัญชี
getOwnedShop()ดึงข้อมูลร้านค้าที่ผู้ใช้เป็นเจ้าของ (null ถ้ายังไม่มีร้าน)
getUpgradeRequests()ดึงรายการคำขอเลื่อนขั้นทั้งหมดของ Account นี้ พร้อมสถานะและเหตุผลที่ไม่ผ่าน
getClaimRequests()ดึงรายการคำขอเคลมที่เคยยื่น เพื่อแสดงในหน้าประวัติ
getReportRequests()ดึงรายการรายงานร้านค้าที่เคยส่ง
entity · actor
Admin

เก็บข้อมูลแอดมินของระบบ ต่างจาก Account ตรงที่มี is_active เพื่อระงับสิทธิ์ได้ทันที และมี last_login สำหรับ audit trail

Methods & คำอธิบาย
login()ล็อกอิน admin panel ผ่าน Google OAuth และอัปเดต last_login
logout()ออกจากระบบและ invalidate session
getActivityLogs()ดึง log การทำงานทั้งหมดของแอดมินคนนี้ เพื่อตรวจสอบย้อนหลัง
entity · core
Shop

หัวใจหลักของระบบ เก็บข้อมูลร้านค้า shop_status (ขั้น 1/2/3), is_blacklist และ failed_upgrade_count ที่ใช้จำกัดการขอเลื่อนขั้นรายเดือน

Methods & คำอธิบาย
create()สร้างร้านค้าใหม่ในระบบ เชื่อม owner_account_id กับ Account ที่เป็นเจ้าของ
update()แก้ไขข้อมูลร้าน เช่น ชื่อ URL ช่องทางติดต่อ ตรวจ format ก่อนบันทึก
delete()Soft delete — ตั้ง is_active = false ให้ร้านหายจากผลค้นหา ข้อมูลยังอยู่ใน DB
search()ค้นหาร้านจาก keyword พร้อม pagination กรอง is_active = true
getDetail()ดึงข้อมูลรายละเอียดเต็มสำหรับแสดงหน้า detail รวม channel และ status
setBlacklist()ตั้ง is_blacklist = true และสร้าง record ใหม่ใน Blacklist table พร้อม adminId
upgradeToTier()อัปเดต shop_status ไปยังขั้นที่กำหนด (1→2 หรือ 2→3) หลังแอดมินอนุมัติ
getBlacklistHistory()ดึงประวัติ blacklist ทั้งหมดของร้านนี้ รวมทั้งที่ยกเลิกไปแล้ว
getUpgradeRequests()ดึงรายการคำขอเลื่อนขั้นทั้งหมดของร้านนี้ พร้อม admin_remark
Request entities — คำร้องต่างๆ
entity · request
UpgradeRequest

คำขอเลื่อนขั้นร้านค้าจากขั้น 1→2 มี admin_remark เพื่อบันทึกเหตุผลกรณีไม่อนุมัติ ให้ร้านค้ารู้ว่าต้องแก้อะไร

Methods & คำอธิบาย
submit()ร้านค้ายื่นคำขอพร้อมเอกสาร ตรวจเงื่อนไข failed_upgrade_count ก่อน บันทึก status = pending
approve()แอดมินอนุมัติ → เปลี่ยน status = approved เรียก Shop.upgradeToTier() และบันทึก UpgradeApprovalLog
reject()แอดมินปฏิเสธ → บันทึกเหตุผลใน admin_remark เพิ่ม Shop.failed_upgrade_count และบันทึก Log
getAttachments()ดึงไฟล์เอกสารทั้งหมดที่แนบมาพร้อมคำขอนี้
getApprovalLogs()ดึง log การอนุมัติ/ปฏิเสธทั้งหมดของคำขอนี้ เพื่อแสดงประวัติ
entity · request
ClaimRequest

คำขอเคลมของผู้ใช้ที่ถูกโกง เก็บ contact_info เพื่อให้แอดมินติดต่อกลับ และ evidence_url อ้างอิงหลักฐาน

Methods & คำอธิบาย
submit()ผู้ใช้ยื่นคำขอพร้อมรายละเอียดและเอกสาร บันทึก status = open รอแอดมินตรวจสอบ
close()แอดมินปิดคำขอเมื่อติดต่อผู้ใช้และชดเชยเรียบร้อยแล้ว เปลี่ยน status = closed
getAttachments()ดึงไฟล์หลักฐานทั้งหมดที่ผู้ใช้แนบมาในคำขอนี้
entity · request
ReportRequest

รายงานร้านค้าต้องสงสัยโดยผู้ใช้ เก็บ damage_amount ประเมินความเสียหาย และเชื่อมกับ Blacklist ได้ถ้าแอดมินตัดสินใจ blacklist

Methods & คำอธิบาย
submit()ผู้ใช้ส่งรายงานพร้อมเหตุผลและหลักฐาน บันทึก status = pending รอแอดมินตรวจสอบ
process()แอดมินดำเนินการเสร็จ เปลี่ยน status = done อาจนำไปสู่การสร้าง Blacklist record ต่อ
getAttachments()ดึงเอกสารหลักฐานทั้งหมดที่ผู้ใช้ส่งมาพร้อมรายงาน
Support entities — ระบบสนับสนุน
entity · support
Attachment

ไฟล์เอกสารที่แนบกับคำร้องทุกประเภท ใช้ร่วมกันได้ทั้ง UpgradeRequest, ClaimRequest และ ReportRequest ระบบ log ทุกครั้งที่แอดมินดาวน์โหลด

Methods & คำอธิบาย
upload()รับไฟล์จาก client, บันทึก URL และเชื่อม foreign key กับคำร้องที่ระบุ (upgrade/claim/report)
download()คืน signed URL หรือ stream ไฟล์ให้แอดมิน พร้อมเรียก logDownload() อัตโนมัติ
logDownload()บันทึกว่าแอดมินคนไหน (adminId) ดาวน์โหลดเมื่อไหร่ สำหรับ audit trail
delete()ลบไฟล์ออกจากระบบ ใช้กรณีคำร้องถูกยกเลิกหรือลบข้อมูล
entity · support
Blacklist

บันทึกสถานะ blacklist ของร้านค้า มี is_active รองรับการยกเลิก blacklist ในอนาคต เชื่อมกับ ReportRequest ที่เป็นต้นเหตุ

Methods & คำอธิบาย
create()สร้าง record blacklist ใหม่ พร้อมอัปเดต Shop.is_blacklist = true ทันที
deactivate()ยกเลิก blacklist ตั้ง is_active = false และอัปเดต Shop.is_blacklist = false กลับ
getHistory()ดึงประวัติ blacklist ทั้งหมดของร้านนั้น รวมทั้งที่ยกเลิกแล้ว เพื่อแสดงพฤติกรรมในอดีต
entity · support
UpgradeApprovalLog

บันทึก audit trail ทุก action ของแอดมินที่เกี่ยวกับการอนุมัติเลื่อนขั้น ช่วยตรวจสอบย้อนหลังว่าใครทำอะไร เมื่อไหร่

Methods & คำอธิบาย
create()บันทึก log ทันทีที่แอดมินอนุมัติหรือปฏิเสธ เก็บ action, reason และ timestamp
getByAdmin()ดึง log ทั้งหมดที่แอดมินคนนั้นทำ เพื่อตรวจสอบการทำงาน
getByShop()ดึงประวัติการอนุมัติ/ปฏิเสธทั้งหมดของร้านค้านั้น แสดงในหน้า MyShopPage
